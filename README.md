## What is HireOS?

HireOS automates technical interviewing end-to-end. HR teams configure a multi-round pipeline once — voice screen, live coding, system design, online assessment — then send candidates a link. From that point, an AI agent conducts the interview, evaluates responses in real time, monitors integrity, and delivers a scored report to the hiring team before the candidate has closed their laptop.

### Key capabilities

- **AI voice interviewer** — Conducts natural spoken interviews with sub-800ms response latency. Adapts question difficulty in real time based on candidate performance using exponential moving average scoring.
- **Live coding evaluation** — Monaco editor with real-time code analysis. The AI detects brute-force approaches, missing edge cases, and copy-paste patterns, then asks targeted follow-up questions through the same voice channel.
- **System design whiteboard** — Excalidraw-powered canvas with AI discussion over voice.
- **Online assessment** — MCQ and coding problems graded automatically. Code runs in an isolated gVisor sandbox with no network access.
- **Automated evaluation** — Four-dimension real-time scoring (technical accuracy, depth, clarity, conciseness) with a comprehensive final report generated by Anthropic Claude. Includes anti-AI-assistance signal detection.
- **Session recovery** — Every interview is fully resumable. Candidates who lose their connection are back in the interview within 30 seconds, continuing from where they left off.
- **Non-blocking proctoring** — Camera-based face detection runs in a Web Worker. Integrity score starts at 100 and is deducted per incident. Never freezes the interview UI.

## Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                    Candidate Browser                            │
│  Next.js Portal  ←WebSocket→  Cloudflare Worker (DO)           │
└──────────────────────────┬──────────────────────────────────────┘
                           │ Redis Streams
┌──────────────────────────▼──────────────────────────────────────┐
│                 Interview WS Server (Fastify :3001)             │
│                                                                 │
│   ConversationAgent   CodeWatcherAgent                          │
│   EvaluationAgent     ProctorAgent                              │
│                                                                 │
│   LLMRouter (CF AI → Anthropic)   STTService (Deepgram)        │
│   TTSService (ElevenLabs/Azure)   CodeExecutionService          │
│   AdaptiveDifficultyEngine        TranscriptNormalizer          │
└──────────────────────────┬──────────────────────────────────────┘
                           │
         ┌─────────────────┴──────────────────┐
         ▼                                    ▼
  Code Sandbox (Docker + gVisor :4000)   Cloudflare R2
  Python · Node · Java · Go · C++        (recordings)
```

**LLM routing strategy:** Cloudflare Workers AI (Llama 3.1) handles fast, latency-sensitive decisions. Anthropic Claude handles final evaluation where quality matters more than speed. If both fail, the interview continues with graceful fallback behavior — the candidate is never blocked.

| Layer | Technology |

## Getting Started

### Prerequisites

- Node.js 18+
- Docker (for Redis and code sandbox)
- A [Supabase](https://supabase.com) project
- A [Cloudflare](https://dash.cloudflare.com) account (Workers, R2, AI)
- API keys for [Deepgram](https://deepgram.com), [ElevenLabs](https://elevenlabs.io), and [Anthropic](https://anthropic.com)

### 1. Install dependencies

```bash
npm install
cd server && npm install
cd cloudflare && npm install
```

### 2. Configure environment variables

```bash
cp .env.local.example .env.local
```

Edit `.env.local` with your credentials (see [Environment Variables](#environment-variables) below).

### 3. Run database migrations

In your Supabase project's SQL editor, run in order:

```
supabase/migrations/001_initial_schema.sql
supabase/migrations/002_part2_interview_engine.sql
```

Or use the helper script as a reminder:

```bash
npm run db:migrate:002
```

### 4. Start services

```bash
# Start Redis + code sandbox (Docker)
docker-compose up -d redis sandbox

# Start Next.js and the interview WS server together
npm run dev:all
```

| Service | URL |
|---|---|
| Next.js app | http://localhost:3000 |
| Interview WS server | ws://localhost:3001 |
| Code sandbox | http://localhost:4000 |

### 5. Deploy Cloudflare Worker

```bash
npm run deploy:worker
```

Update `CLOUDFLARE_WORKER_URL` in `.env.local` with the deployed URL.

## Database Schema

10 tables total, all with row-level security.

| Table | Description |
|---|---|
| `organizations` | Multi-tenant root — one org per account |
| `profiles` | User profiles linked to Supabase Auth |
| `jobs` | Job postings with `pipeline_config` (JSONB) and `scoring_rubric` (JSONB) |
| `candidates` | Applicants linked to a job and org |
| `candidate_invites` | Signed portal tokens with expiry and status |
| `round_results` | Per-round evaluation data, transcript, scores, AI analysis |
| `interview_checkpoints` | Session recovery state (90-second snapshots) |
| `recording_files` | R2 file references with presigned URL expiry |
| `oa_problems` | OA question bank (MCQ and coding, org-specific and shared) |
| `proctor_events` | Interview integrity event log |

## Security

- All candidate code runs inside a **gVisor (runsc) sandbox** with `--network=none`, dropped capabilities, and 256MB memory limit. Code never executes on the host.
- Row-level security is enforced on every table. The service role key is never exposed to the browser.
- Invite tokens are validated server-side on every portal request.
- The AI interviewer's kindness and professionalism instruction is global and cannot be overridden by HR's custom prompt configuration.
- Security headers (`X-Frame-Options`, `X-Content-Type-Options`, `Referrer-Policy`, CSP) are set in `next.config.ts`.
